# ARCHITECTURE.md

## اسکیمای دیتابیس
- **وضعیت:** N/A (پروژه فعلی یک لندینگ‌پیج استاتیک/کلاینت‌ساید است و نیازی به اتصالات دیتابیس در این فاز ندارد). دیتای داینامیک مانند پلن‌های قیمت‌گذاری و نظرات کاربران به صورت استاتیک در داخل فایل‌های کامپوننت (آرایه‌های ثابت) مدیریت می‌شوند.

## مسیردهی API و جریان داده
- داده‌ها به صورت استاتیک از طریق ساختارهای داده (Data Structures) موجود در فایل‌های `components/sections/` (مانند آرایه‌های `plans` و `testimonials` و `metrics`) به کامپوننت‌های UI پاس داده می‌شوند. جریان داده کاملاً یک‌طرفه (Top-Down) است.
- مسیردهی صفحات صرفاً از طریق `next/link` برای اسکرول به بخش‌های مختلف (`#pricing` و غیره) انجام می‌پذیرد.

## قوانین درخت فایل (File Tree)
این پروژه از پیش موجود است. منطق مسیردهی و ساختار پوشه‌ها به شرح زیر تثبیت می‌شود:
- **`components/ui/`**: کامپوننت‌های خرد و ایزوله (مثل دکمه‌ها و `navbar.tsx`).
- **`components/sections/`**: بخش‌های کلیدی لندینگ پیج که کانتکست مارکتینگی را در خود جای داده‌اند. ویرایش‌های فعلی تماماً روی فایل‌های موجود در این مسیر انجام می‌شود.

**فایل‌های هدف جهت ویرایش (بدون ایجاد فایل جدید):**
1. `components/ui/navbar.tsx`
2. `components/sections/hero-section.tsx`
3. `components/sections/features-section.tsx`
4. `components/sections/pricing-section.tsx`
5. `components/sections/testimonials-section.tsx`
6. `components/sections/cta-section.tsx`
